# QA Track v0.1.0

> Primeira versão funcional do sistema de gestão de cenários de teste.

---

## Funcionalidades

### Cenários de Teste
- Cadastro de cenários com título único
- Pré-requisitos (múltiplos por cenário)
- Resultados esperados (múltiplos por cenário)
- Vínculo com feature e status
- Associação com múltiplos sistemas (N:N via `/api/system/:id/scenario`)

### Sistemas
- CRUD completo
- Associação e desassociação de cenários

### Features
- CRUD completo
- Listagem por sistema (`/api/feature/system/:systemId`)

### Status de Cenários
- Cadastro de status personalizados
- CRUD completo

### Config
- Export completo do banco (`GET /api/config/export`)
- Import de backup (`POST /api/config/import`)

## Endpoints da API

### Sistemas `/api/system`
| Método | Endpoint | Descrição |
|--------|----------|-----------|
| GET | `/api/system` | Listar todos |
| GET | `/api/system/:id` | Detalhes |
| POST | `/api/system` | Criar |
| PATCH | `/api/system/:id` | Atualizar |
| DELETE | `/api/system/:id` | Remover |
| POST | `/api/system/:id/scenario` | Associar cenário ao sistema |
| DELETE | `/api/system/:id/scenario/:scenarioId` | Desassociar cenário do sistema |

### Features `/api/feature`
| Método | Endpoint | Descrição |
|--------|----------|-----------|
| GET | `/api/feature` | Listar todas |
| GET | `/api/feature/:id` | Detalhes |
| GET | `/api/feature/system/:systemId` | Listar por sistema |
| POST | `/api/feature` | Criar |
| PATCH | `/api/feature/:id` | Atualizar |
| DELETE | `/api/feature/:id` | Remover |

### Cenários `/api/scenario`
| Método | Endpoint | Descrição |
|--------|----------|-----------|
| GET | `/api/scenario` | Listar todos |
| GET | `/api/scenario/:id` | Detalhes completos |
| POST | `/api/scenario` | Criar |
| PATCH | `/api/scenario/:id` | Atualizar |
| DELETE | `/api/scenario/:id` | Remover |
| POST | `/api/scenario/:id/pre` | Adicionar pré-requisito |

| DELETE | `/api/scenario/pre/:preId` | Remover pré-requisito |
| POST | `/api/scenario/:id/expect` | Adicionar resultado esperado |
| PATCH | `/api/scenario/expect/:expectId` | Atualizar resultado esperado |

## Modelo de Dados

```
t_system (id, title)
    └── t_feature (id, title, system_id)
            └── t_scenario (id, title, status_id, feature_id)
                    ├── t_scenario_system (scenario_id, system_id)  ← N:N com t_system
                    ├── t_scenario_pre (id, scenario_id, description)
                    └── t_scenario_expect (id, scenario_id, description)

t_scenario_status (id, title)  ← referenciado por t_scenario.status_id
```
